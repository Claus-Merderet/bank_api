# Bank API Frontend

## What This Is

Фронтенд (SPA на React + Vite) для учебного банковского API на Symfony. Бэкенд — готовый полигон, на котором тестировщики учатся тестировать; фронтенд делает его функциональность доступной через UI: авторизация, админка, банковские счета, пополнения, переводы, кредиты, история транзакций. Код фронтенда живёт в `frontend/` внутри этого репозитория.

## Core Value

UI покрывает все ручки бэкенда и содержит все обязательные UI-элементы из требований к работе — чтобы тестировщикам было что тестировать.

## Requirements

### Validated

<!-- Существующие возможности бэкенда (из карты кодовой базы) — фронт строится поверх них. -->

- ✓ JWT-аутентификация (login по username/password, lexik/jwt) — existing
- ✓ Админ: создание пользователей с ролями (ROLE_USER, ROLE_CREDIT_SECRET), список всех пользователей, удаление по id (soft delete) — existing
- ✓ Счета: создание (макс. 2 на пользователя), пополнение (deposit 1000–9000), переводы между своими счетами и другим пользователям (500–10000), история транзакций (deposit / transfer_in / transfer_out) — existing
- ✓ Кредиты (только ROLE_CREDIT_SECRET): запрос 5000–15000 на один из счетов, погашение только один раз и на всю сумму, второй кредит при активном/на другом счёте невозможен — existing
- ✓ Валидация кредов: username — латиница, 3–15 символов, буквы и цифры без спецсимволов; password — латиница, от 8 символов, минимум 1 заглавная, 1 строчная, 1 цифра, 1 спецсимвол — existing
- ✓ Дефолтный админ: username `admin`, password `123456` — existing

### Out of Scope

- Изменения бэкенда — бэк готовый и учебный, трогать его нельзя (кроме, при необходимости, CORS-конфига для работы SPA)
- Автотесты фронтенда — проект учебный, тестировать его будут тестировщики вручную
- Адаптив под мобильные — не требуется заданием; десктоп-first
- Регистрация пользователей самим пользователем — по требованиям юзеров создаёт только админ

## Context

**Рабочий флоу проекта (важно):**
1. Claude подробно расписывает все существующие ручки API, их взаимодействия и механики → документ для дизайна
2. Пользователь берёт это описание в десктопный Claude и делает дизайнерские макеты
3. Пользователь скидывает скриншоты макетов в сессию
4. Claude реализует фронтенд по макетам

**Бэкенд** (карта: `.planning/codebase/`):
- Symfony + Doctrine + PostgreSQL, JWT (lexik), Nelmio API Doc (Swagger)
- Контроллеры: `src/Controller/AccountController.php`, `AdminController.php`, `CreditController.php` + login через security (json_login / lexik)
- Запуск через Docker (`docker-compose.yml`, nginx + php-fpm)
- Роли: ROLE_ADMIN, ROLE_USER, ROLE_CREDIT_SECRET
- При удалении юзера информация не удаляется физически (soft delete): счета, история, кредиты сохраняются

**Исходные требования**: скриншоты в папке `Требования/` (1.png, 2.png, 3.png) рядом с репозиторием.

## Constraints

- **Стек**: React + Vite — выбор пользователя; без TypeScript-жёсткости, если не потребуется
- **Расположение**: `frontend/` внутри репозитория bank_api
- **Бэкенд неприкосновенен**: фронт подстраивается под существующие ручки, а не наоборот
- **Дизайн**: реализация строго по макетам, которые пользователь принесёт из десктопного Claude
- **Обязательные UI-элементы**: 9 пунктов из требований к работе — это критерий приёмки
